# Joys Fonts

A modern platform and CLI tool to package and install Persian web fonts in Tailwind CSS projects (specifically Tailwind CSS v4+) without manual configuration or installing the entire repository as a dependency. The library is distributed as a Node.js package with a built-in CLI (`npx joys-fonts add <font-slug>`) that copies only the selected font assets directly to the consumer's workspace.

## Project Structure

```
joys-fonts/
├── bin/
│   └── cli.js              (installer CLI entry point)
├── src/
│   └── index.js            (programmatic API entry point)
├── fonts/
│   └── joys-<font-name>/
│       ├── joys-<font-name>.css
│       └── woff2/
│           └── joys-<font-name>[-<variant>]-<weight>.woff2
├── fonts.manifest.json     (auto-generated database index — do not edit manually)
├── generate-manifest.js    (development tool to scan fonts/ and build manifest)
├── index.html              (development tool for visual inspection of fonts)
├── package.json            (npm package configuration)
├── .npmignore              (rules to exclude dev files from npm package)
└── .AGENTS.md              (project documentation)
```

**Important Note:** Never hardcode any specific font name in index.html, generate-manifest.js, or code files. The single source of truth for the available fonts is `fonts.manifest.json`, which is auto-generated from the `fonts/` directory.

## Naming & Formatting Rules

- Every font must have a dedicated directory under `fonts/` named `joys-<font-name>` in kebab-case (e.g., `joys-vazir-matn`).
- Only the **WOFF2** format is kept (no outdated ttf/otf/eot/woff formats) since this is exclusively for modern web use.
- If the font has parallel variants:
  - `FaNum` (Persian numbers) → suffixed with `-fanum` (e.g., `<slug>-fanum-<weight>.woff2`)
  - `NoEn` (No English letters) → suffixed with `-noen` (e.g., `<slug>-noen-<weight>.woff2`)
- For **Variable** Fonts:
  - Use `<slug>-vf.woff2` (or `<slug>-<variant>-vf.woff2`).
  - In CSS `@font-face`, use range-based font-weight: `font-weight: min max;`.
  - If it supports the width axis (`wdth`), use range-based font-stretch: `font-stretch: min% max%;`.
  - Always check axis details using font inspectors (like wakamaifondue.com) or vendor docs before building the CSS, since default values of some fonts can be unexpected.
- For **Static** Fonts:
  - Each weight must have a separate `@font-face` declaration with standard numeric CSS weights.
  - Standard weight mappings: Thin=100, ExtraLight/UltraLight=200, Light=300, Regular=400, Medium=500, SemiBold/DemiBold=600, Bold=700, ExtraBold=800, Black=900.
  - Non-standard weights above 900 (Heavy, ExtraBlack, etc.) should map to a logical estimate (e.g., 950 or 1000) and be noted in the CSS.

## Each Font Directory Must Contain

1. `woff2/` — only the woff2 files renamed according to the naming rules.
2. `<slug>.css` — including:
   - A `@font-face` definition for every weight/variant combination.
   - A base CSS class `.font-<slug>[-<variant>]` setting `font-family`, a default `font-weight: 400;`, and default `font-stretch: 100%;` if a width axis is present.
   - Separate semantic classes for OpenType feature settings (e.g. stylistic sets ss01, ss02) using clear class names (not `.ss01`).

## Workflow for Adding a New Font

1. Acquire the font files. Keep only the `woff2` versions (convert using tools like python's `fontTools` or online converters like transfonter.org if only `ttf`/`otf` are available).
2. Create the directory: `fonts/joys-<name>/woff2/`.
3. Rename the font files according to the naming rules.
4. Write `<slug>.css` (you can copy from an existing similar font and customize weights/families).
5. Generate the database manifest:
   ```bash
   node generate-manifest.js
   ```
6. Start a local server:
   ```bash
   npx -y http-server -p 8000
   ```
7. Open `http://localhost:8000` to visually verify the weights, width axes (if variable), and stylistic features.

## Development Scripts

- `generate-manifest.js` — Scans `fonts/` and builds `fonts.manifest.json`. Fonts that do not have a CSS file yet are skipped and reported in the console.
- `index.html` — A visual testing page that is fully data-driven from `fonts.manifest.json`.

## Installer CLI Details

The CLI script `bin/cli.js` allows users to install any font locally:
```bash
# Run locally during development:
node bin/cli.js add <font-slug>

# Or run using a package manager once published:
npx joys-fonts add <font-slug>
pnpm dlx joys-fonts add <font-slug>
yarn dlx joys-fonts add <font-slug>
bunx joys-fonts add <font-slug>
```
The CLI will copy the font assets, inject `@import` statements to the target CSS, and automatically register Tailwind CSS v4 variables in the `@theme` block.
